### Run a model

```
zerollama run gemma3
```

### Launch integrations

```
zerollama launch
```

Configure and launch external applications to use Ollama models. This provides an interactive way to set up and start integrations with supported apps.

#### Supported integrations

- **OpenCode** - Open-source coding assistant
- **Claude Code** - Anthropic's agentic coding tool
- **Codex** - OpenAI's coding assistant
- **VS Code** - Microsoft's IDE with built-in AI chat
- **Droid** - Factory's AI coding agent

#### Examples

Launch an integration interactively:

```
zerollama launch
```

Launch a specific integration:

```
zerollama launch claude
```

Launch with a specific model:

```
zerollama launch claude --model qwen3.5
```

Configure without launching:

```
zerollama launch droid --config
```

#### Multiline input

For multiline input, you can wrap text with `"""`:

```
>>> """Hello,
... world!
... """
I'm a basic program that prints the famous "Hello, world!" message to the console.
```

#### Multimodal models

```
zerollama run gemma3 "What's in this image? /Users/jmorgan/Desktop/smile.png"
```

### Generate embeddings

```
zerollama run embeddinggemma "Hello world"
```

Output is a JSON array:

```
echo "Hello world" | zerollama run nomic-embed-text
```

### Download a model

```
zerollama pull gemma3
```

### Remove a model

```
zerollama rm gemma3
```

### List models

```
zerollama ls
```

### Sign in to Ollama

```
zerollama signin
```

### Sign out of Ollama

```
zerollama signout
```

### Create a customized model

First, create a `Modelfile`

```
FROM gemma3
SYSTEM """You are a happy cat."""
```

Then run `zerollama create`:

```
zerollama create -f Modelfile
```

### List running models

```
zerollama ps
```

### Stop a running model

```
zerollama stop gemma3
```

### Start Ollama

```
zerollama serve
```

To view a list of environment variables that can be set run `zerollama serve --help`
